# Product Philosophy

AspidaSec should be a focused security product, not a collection of security ideas.

The product exists to make website security findings understandable, prioritized, and fixable for developers.

## Principles

### 1. Evidence Before Explanation

Scanner evidence comes first. Explanations, summaries, and fix guidance are only useful when attached to a real finding.

### 2. Fix Clarity Beats Feature Count

A short report that helps a developer fix one real vulnerability is more valuable than a large report that nobody acts on.

### 3. Prioritization Is Product Value

AspidaSec should not only list findings. It should help the user decide what to fix first.

Signals that matter:

- Severity.
- Confidence.
- Public exposure.
- Exploitability.
- Affected endpoint or file.
- Production relevance.
- Dependency directness.
- Fix availability.

### 4. CLI First, Dashboard Later

The CLI and report workflow must feel excellent before hosted dashboards, authentication, analytics, or collaboration systems are built.

### 5. Localized, Not Simplified

Bahasa Indonesia guidance should preserve security meaning. Localization should make findings clearer without weakening technical accuracy.

### 6. Open Source Should Be Useful Alone

The open-source CLI, scanner engine, report templates, and basic guidance should be valuable without requiring a hosted service.

### 7. AI Supports Remediation, Not Detection

AI can summarize and explain. It cannot invent findings, evidence, severity, affected files, or affected endpoints.

## Product Voice

AspidaSec should sound:

- Direct.
- Practical.
- Evidence-aware.
- Developer-friendly.
- Security-conscious.

AspidaSec should avoid sounding:

- Academic.
- Fear-driven.
- Magical.
- Overly broad.
- Like an AI playground.

## Messaging Rule

Lead with:

> Website security scanning with practical remediation guidance.

Avoid leading with:

- AI cybersecurity ecosystem.
- Developer education platform.
- Universal security scanner.
- Multi-agent security lab.
- Gamified security learning.

## Product Bet

Many scanners already exist. AspidaSec wins only if the output is easier to act on:

- Less noise.
- Better grouping.
- Better prioritization.
- Better remediation.
- Better localization.
- Better reports.

That is the product moat.
