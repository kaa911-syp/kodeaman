# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in KodeAman, please report it responsibly.

**DO NOT** open a public GitHub issue for security vulnerabilities.

### How to Report

Email: **security@kodeaman.dev**

Please include:

1. Description of the vulnerability
2. Steps to reproduce
3. Potential impact
4. Suggested fix (if any)

### Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial assessment**: Within 5 business days
- **Resolution target**: Within 30 days for critical issues

### Scope

The following are in scope:

- KodeAman CLI (`@kodeaman/cli`)
- KodeAman GitHub Bot (`@kodeaman/bot-github`)
- KodeAman GitLab Bot (`@kodeaman/bot-gitlab`)
- All `@kodeaman/*` packages
- Official Docker images

### Out of Scope

- Third-party scanner vulnerabilities (report to Semgrep/ZAP directly)
- Social engineering attacks
- Denial of service attacks

## Supported Versions

| Version | Supported |
|---------|-----------|
| 0.1.x   | ✅ Current |

## Recognition

We appreciate responsible disclosure and will credit reporters (with permission) in our release notes.
